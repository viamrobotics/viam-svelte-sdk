---
title: Debugging
description: Inspect SDK activity with the built-in logger and live console controls.
---

The SDK uses a built-in logger that writes all messages (including query/mutation request, response, and error events) to the browser console and an in-memory buffer.

By default the console output is enabled at the `info` level. The log buffer always captures all levels regardless of the console setting.

**Available log levels** (from `SDKLogLevel`): `trace`, `debug`, `info`, `warn`, `error`, `fatal`.

Setting the level to `trace` also enables the TypeScript SDK's built-in gRPC trace logging, which logs every unary and streaming gRPC request/response to the console via `console.trace` and `console.debug`. You must refresh the page after setting the level to enable/disable the trace output.

## Setting the log level from the provider

Pass `logLevel` to [`ViamProvider`](../getting-started/) to set the default for the app:

```svelte
<script lang="ts">
import { ViamProvider, SDKLogLevel } from '@viamrobotics/svelte-sdk';
import type { DialConf } from '@viamrobotics/sdk';

let { children } = $props();

const dialConfigs: Record<string, DialConf> = { /* ... */ };
</script>

<ViamProvider
  {dialConfigs}
  logLevel={SDKLogLevel.debug}
>
  {@render children()}
</ViamProvider>
```

The provider checks `localStorage` first and prefers any level the user has set via the browser console (see below). The `logLevel` prop is the fallback default.

## Changing the level from the browser console

The level set via `window.VIAM.setSDKLogLevel` is persisted in `localStorage` and restored on every page load. This always wins over the provider's `logLevel` prop, so it's safe to give to a customer for live debugging without redeploying.

```js
// Change the log level (persisted across refreshes)
window.VIAM.setSDKLogLevel('debug');
window.VIAM.setSDKLogLevel('info');

// Silence the console (persisted across refreshes; logs still accumulate in the buffer)
window.VIAM.setSDKLogLevel(false);

// Reset to the default info level
window.VIAM.setSDKLogLevel('info');

// Retrieve all buffered log entries (up to 10,000)
window.VIAM.getSDKLogs();

// Clear the log buffer
window.VIAM.clearSDKLogs();
```
